<p>My last skeletal was created with major elements grouped, but separately, then moved around in PS. Thus, the skeletal is very mobile. I actually keep PSD file versions of a lot of my figure work, allowing me to adapt the material as needed.<br><br>The last time I tried multi-view skeletals, it was one of <em>Scleromochlus taylori</em> in lateral, dorsal, cranial and posterior views, and this was done on multiple nested layers, in order to make sure the elements lined up. That was a project of weeks, as it was all rendered by hand, each view.<br><br>The problem, though, is sources. 3D illustrations require either on-hand manipulation or otherwise 3D images to be able to view and render illustrations to the appropriate angles. Benton's description of the above taxon gives you the dorsal, ventral, and side views, and everything else was imagination. So it becomes interesting, as I find myself concepting skulls in a "neutral" pose, to find that skulls in anterior view (and "neutral") look nothing like we are accustomed to. This is very hard to find sources to allow you to adjust, unless you can manipulate the objects yourself.</p>
